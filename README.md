### 1. Описание проекта

Данный проект представляет собой веб-калькулятор, разработанный в рамках лабораторной работы №22. Основная цель — создание интерактивного приложения с использованием JavaScript, демонстрирующего принципы работы с DOM, обработку событий, а также применение модульной архитектуры для повышения читаемости, безопасности и поддерживаемости кода.

Калькулятор имеет стандартный интерфейс с кнопками цифр, основных арифметических операций, кнопкой очистки и кнопкой вычисления результата.

### 2. Функциональность

#### • Какие операции поддерживает?

Калькулятор поддерживает базовые арифметические операции:

- **Сложение** (`+`)
- **Вычитание** (`-`)
- **Умножение** (`*`)
- **Деление** (`/`)
- Ввод целых и десятичных чисел (с использованием точки).

#### • Какие есть ограничения?

Для обеспечения корректной работы и предотвращения ошибок введены следующие ограничения на ввод:

- **Запрет на множественные операторы:** Нельзя ввести два оператора подряд (например, `5++3` или `2*-/4`).
- **Запрет на оператор в конце:** Выражение не может заканчиваться оператором (например, `5+` не будет вычислено).
- **Запрет на множественные точки:** В одном числе можно поставить только одну десятичную точку (например, `5.2.1` недопустимо).

#### • Как обрабатываются ошибки?

- **Валидация:** При вводе некорректного выражения (нарушение ограничений выше) калькулятор не позволяет добавить ошибочный символ, сохраняя текущее корректное состояние.
- **Вычисление:** Для вычисления выражения используется безопасный метод `safeEvaluate`.
  - При попытке вычислить некорректное выражение (например, деление на ноль) на дисплее отображается сообщение **`Ошибка`**.
  - После возникновения ошибки калькулятор блокируется до нажатия кнопки очистки (`C`), что предотвращает дальнейшие некорректные действия.

#### • Описать класс Calculator

Класс `Calculator` — это ядро приложения. Он инкапсулирует в себе всё состояние и поведение калькулятора, изолируя логику от DOM-манипуляций, что упрощает тестирование и модификацию.

**Основные поля класса:**

- `displayElement`: Ссылка на DOM-элемент поля ввода (дисплей).
- `expression`: Строка, хранящая текущее математическое выражение.
- `hasError`: Флаг (boolean), указывающий, находится ли калькулятор в состоянии ошибки.

**Основные методы класса:**

- `init()`: Инициализирует калькулятор, навешивая обработчики событий на кнопки.
- `handleButtonClick(value)`: Главный маршрутизатор, который определяет, что нужно сделать при нажатии кнопки (очистить, вычислить или добавить символ).
- `safeEvaluate(expr)`: Безопасно вычисляет переданное выражение, используя `eval` с предварительной валидацией и обработкой ошибок.
- `isValidExpression()`: Проверяет корректность текущего выражения (запрет на два оператора подряд, оператор в конце).
- `updateDisplay()`: Обновляет отображение на экране калькулятора.

#### • Схема взаимодействия компонентов

1.  **Пользователь** нажимает на кнопку в `index.html`.
2.  Срабатывает событие `click`, обработчик которого был настроен в `main.js` при вызове метода `init()` класса `Calculator`.
3.  Управление передается в метод `handleButtonClick()` класса `Calculator`, который находится в файле `calculator.js`.
4.  `Calculator` изменяет свое внутреннее состояние (поле `expression`) и вызывает метод `updateDisplay()`.
5.  `updateDisplay()` обновляет текст в DOM-элементе дисплея.
6.  Пользователь видит обновленный результат на странице.

### 4. Логика работы

#### • Последовательность действий при нажатии кнопок

1.  **Нажатие на цифру или оператор (+, -, \*, /):**
    - Вызывается `handleButtonClick`.
    - Проверяется состояние ошибки (`hasError`). Если есть ошибка, и нажата не `C`, калькулятор сначала очищается.
    - Происходит проверка на допустимость ввода (валидация).
    - Если ввод допустим, символ добавляется к строке `expression`.
    - Вызывается `updateDisplay()` для отображения обновленного выражения.

2.  **Нажатие на кнопку `C` (Очистка):**
    - Вызывается метод `clear()`.
    - Строка `expression` сбрасывается в пустую строку.
    - Флаг `hasError` устанавливается в `false`.
    - Дисплей обновляется, показывая `0` (по умолчанию).

3.  **Нажатие на кнопку `=` (Равно):**
    - Вызывается `handleButtonClick` для кнопки `=`.
    - Вызывается метод `safeEvaluate(expression)`.
    - **Если валидация пройдена:** Вычисляется результат. Результат преобразуется в строку, сохраняется в `expression` и отображается на дисплее.
    - **Если валидация НЕ пройдена или возникает ошибка вычисления:** Вызывается `showError()`, которая устанавливает `hasError = true` и отображает на дисплее сообщение `"Ошибка"`.

#### • Как работает валидация ввода?

В классе `Calculator` реализованы специализированные методы-валидаторы, которые вызываются перед добавлением нового символа:

1.  **`isOperator(char)`:** Проверяет, является ли символ оператором (`+`, `-`, `*`, `/`).
2.  **`isLastCharOperator()`:** Проверяет, является ли последний символ в текущем выражении оператором. Это используется для предотвращения ввода двух операторов подряд.
3.  **`hasDuplicateDot()`:** Проверяет, есть ли уже точка в текущем вводимом числе. Это предотвращает ввод нескольких точек.

Если пользователь пытается ввести символ, который нарушает одно из правил (например, ставит оператор после оператора), метод `handleButtonClick` просто игнорирует это действие, не изменяя `expression`.

#### • Как происходит вычисление?

Для вычисления используется метод `safeEvaluate(expr)`:

1.  **Очистка от лишних пробелов:** Убираются все пробелы из строки для корректной обработки.
2.  **Валидация:** Вызывается `isValidExpression()`, который проверяет строку на наличие запрещенных паттернов (двойные операторы, оператор в конце).
3.  **Вычисление:**
    - Если валидация пройдена, используется встроенная функция `eval(expr)`. Она вычисляет математическое выражение, переданное в виде строки.
    - `eval` обернут в конструкцию `try...catch`. Это гарантирует, что при любой ошибке (например, деление на ноль) программа не "упадет", а корректно перейдет в блок `catch`.
4.  **Возврат результата:**
    - При успехе возвращается числовой результат.
    - При ошибке (в `catch`) выбрасывается исключение, которое обрабатывается вызывающим кодом, что приводит к отображению сообщения об ошибке на дисплее.

**Важное примечание:** Хотя в данном учебном проекте `eval` используется для простоты, в реальных приложениях его применение не рекомендуется из-за соображений безопасности. В промышленной разработке для вычисления математических выражений используют безопасные парсеры или подключают специализированные библиотеки.
